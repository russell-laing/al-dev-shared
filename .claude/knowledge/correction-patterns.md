# Correction Patterns

Canonical table of recurring correction patterns found during health-loop plan
review. Referenced by `revise-plugin-plan` Phase correction-classification.

## Pattern Table

| Review finding | Canonical correction |
| --- | --- |
| Commit subjects violate convention | `<emoji> type(scope): subject`, full subject ≤72 chars, subject-only (tool project — no body) |
| A task edits the **executor** skill (`implement-plugin-health/SKILL.md`) | Move it **last**, add a restart boundary: stop + re-invoke before Phase 2/3 close-back |
| Bare `git status` can't pass in a dirty worktree | Path-scope every check: `git status --short -- <task-paths>`; forbidden scan over added lines only (`git diff --unified=0`) |
| A task uses `git add -u` or `git add <dir>` in a dirty worktree | Path-scope staging to the exact task paths; for `docs/health/` ledger tasks add a Step 0 that commits any pre-existing dirty health files first, so a later `git add docs/health/` only stages this task's regenerated output. Author Step 0 in the fail-loud conditional form (`if ! git diff --quiet -- <paths> …; then git add …; git commit …; fi`) — never `git add … && git commit … \|\| echo`, whose `\|\| echo` masks genuine commit failures |
| A grep "passes" on pre-existing text | Bound it (frontmatter-only via `awk`, fixed-string `grep -F`, or a strict measurement that exits non-zero) |
| A `… \| grep <id>` check whose success is "no output" | Inverts the exit code — `grep` exits non-zero on no match, so the success case reads as a failed step. Invert it: `if … \| grep -q <id>; then echo "ERROR: still present"; exit 1; fi` |
| A conditional task branch is only partially implemented (e.g. "use `-compare` if materially different" but only the default branch has steps) | Resolve the fork at plan time (rubber-duck has the live evidence) or surface it at the revise user gate; the chosen branch must have its full steps spelled out — no unimplemented branches left in the task |
| A task changes a skill's first description sentence | Regenerate + stage the derived `docs/maintainer_tooling.md` |
| A task is dropped (re-dispositioned) | Renumber remaining tasks contiguously; update the Goal count and Provenance range |
| Spec content uses `YYYY-MM-DD` as path-pattern placeholders (e.g., in JSON schema examples) — the plan's own forbidden-pattern scan then catches these intentional examples | Replace `YYYY-MM-DD` with `<date>` in the spec content's path-pattern examples |
| A plan edits shared-surface files (`profile-al-dev-shared/`) and closes ledger events on task-local `grep`/`wc` checks alone, never running the repo's shared-surface validators | Add a validation-gate task after the last source edit (and any projection regen), before ledger close-back: chain `validate_harness_neutrality.py`, `validate-lens-agents.py`, `validate-knowledge-quality.py`, `python3 scripts/tests/test_generate_agent_projections.py` with `&&` so any failure halts the plan (under Python 3.13/macOS with a libexpat/dynamic-library conflict, substitute the inline-exec fallback from CLAUDE.md "Known Environment Issues" for the test invocation); the task closes no events (`closes_event_ids: []`) |
| An acceptance step runs a validator/test that does **not** actually exercise the task's deliverable — a false proof of done (e.g. asserting "skill conforms to `{verb}-{object}-{aspect}`" via `test_naming_convention.py`, which only checks lens-AGENT names and directory kebab-case) | Replace or supplement with a check that asserts the task's specific outcome: grep the changed contract for the exact removed/added tokens and assert the new artifact matches the rule the task implements; keep the generic validator only for the narrow sub-check it genuinely performs |
| A rename/refactor commit uses `git add -A`/`git add <dir>` and a multi-line body with an AI footer in a **tool** repo | Path-scope staging to the task's exact files, fail loudly if anything outside the allowlist (e.g. `.dev/health-loop-state.md`) is staged; commit subject-only (no body, no `Co-Authored-By`), ≤72 chars, with the canonical `🚚 move` emoji for renames |
| A decline/close ledger-action template (or a task editing one) omits `--closes-event-ids`, though the live storage contract requires every superseding (`fixed`/`declined`/`grandfathered`) event to set it | Add `--closes-event-ids <accepted_event_id>` to the command and state closure is by object/key match **plus** the explicit closure id (omitting it raises an ambiguous-closure integrity warning); update the task's verification grep to prove the flag is present |
| A task's exact-path `git add <p1> <p2>` is followed by plain `git commit` with no check that the index contains only those paths (pre-existing staged paths from unrelated work would be swept in) | Insert a staged-set assertion between `git add` and `git commit`: `actual=$(git diff --cached --name-only \| sort); expected=$(printf '%s\n' '<p1>' '<p2>' \| sort); if [ "$actual" != "$expected" ]; then echo "ERROR: staged set mismatch. Staged: $actual" >&2; exit 1; fi` — for `git add <dir>` tasks (e.g. `docs/health/`), use a scope-leak guard instead: `git diff --cached --name-only \| grep -v '^docs/health/' \| grep . && { echo "ERROR: non-health paths staged" >&2; exit 1; } \|\| true` |
| A task's accepted disposition evidence says to add X (e.g. an approval gate) but the plan intentionally omits X for architectural reasons (e.g. inline-dispatch callers can't tolerate a pause) | Add an explicit supersession note in the task description stating what the accepted evidence said, why the plan intentionally narrows it, and that the close-back records this override — do NOT silently contradict the accepted evidence |
| A token-saving fast path (e.g. a dimension gate that skips derived-context building) drops a *non-derived* manifest section a downstream lens still needs (e.g. the `## --since scoped file lists` section that scopable lenses depend on) | Keep the optimization but require the fast path to still write the non-derived sections; the scoped lists are not derived map context. Add a verification grep **bound to the new gate text** (a distinctive phrase in the inserted block), not to the pre-existing section elsewhere in the same file |
| A final/CI test loop swallows failures with `… \|\| echo "FAILED: $t"` and so exits `0` even when a test failed | Accumulate failure state (`failed=1` on any failure) and `exit "$failed"` after the loop, so a wrapper checking only the exit status cannot read a failed run as success; keep the readable per-test echo |
| A deterministic assembler concatenates lens blocks whose `###` headings collide (two lenses share `Structural Conventions Findings`) → markdownlint MD024 no-duplicate-heading on the generated file | Uniquify colliding headings by appending an object-type suffix (`(agents)`/`(skills)`) to every block carrying the duplicate; keep the `<!-- lens: <name> -->` marker for exact lens identity; extend the smoke test to assemble **both** colliding lenses and run markdownlint on the output |
| A lens-JSON glob is dimension-unaware, so a same-date out-of-dimension JSON left by an interrupted/cleanup-failed run gets assembled into a single-dimension findings file | Make the loader dimension-aware: derive the allowed lens-name prefix set from the requested `--dimensions` and skip any same-date JSON outside it (ignore silently — preserves the "leftover scratch files are harmless" invariant); apply the same filter to the manual fallback text and add a regression test with one in-dimension + one stale out-of-dimension JSON |
| A batch-decline or queue-consuming step does not explicitly clear or commit the consumed queue file (e.g. `.dev/refuted-skips.json`), leaving it perpetually dirty and re-runnable after the task completes | After the queue-consuming command, rewrite the queue file to `[]` in the same step (e.g. `python3 -c "import pathlib; pathlib.Path('.dev/refuted-skips.json').write_text('[]')"`); extend the task's commit staged-set and exact-match assertion to include the cleared queue file alongside the primary artifacts |
| A task's verification step uses a negative grep to assert the NON-PRESENCE of a string that the task itself directs the executor to insert (e.g. a format comment like `# event_id format: disp_YYYYMMDD_NNNNNN`) | Drop the negative grep on intentional format-documentation tokens inserted by the task; keep only positive presence checks on the new text's meaningful distinguishing phrases (e.g. `grep -qF 'JSONL-store fallback'`). If a forbidden-pattern guard is genuinely needed, scope it to the specific unintended form, not the intentional example |
| A task's functional verification asserts that a value appears on a specific JSON output line number (e.g. "field X is on line 2") without accounting for key ordering or pretty-print layout (e.g. `sort_keys=True` shifts the field) | Replace the fragile line-number assertion with a content-presence check: `command \| grep -qF '"field": "expected_value"'` — this tolerates changes in key ordering and pretty-print layout without producing a false-negative on a correctly-implemented command |
| A Markdown validation gate runs the uncached `npx --yes markdownlint-cli` package, which hangs/fails offline (`ENOTFOUND registry.npmjs.org`) | Use `npx --yes markdownlint-cli2` — the package repo-local skills already use and that resolves in the restricted environment — keeping the same file arguments and "0 errors" semantics |
| A plan commits agent source files in Tasks 1–N and defers all projection regeneration to a later standalone task, leaving intermediate commits where source and harness projections disagree (hook-blocking or revert-unsafe) | Move projection regeneration inline: after each agent-source edit and forbidden-pattern scan, run `python3 scripts/generate-agent-projections.py` and extend `git add` to include the matching `claude/`, `copilot/`, and `codex/` projection files for that agent only. Convert the standalone regen task to a verification-only gate (no generator call, no commit). |
| A static-lens verification step writes output to `--out-dir .dev`, leaving untracked JSON artifacts in the worktree that the plan never removes or ignores | Use `OUT_DIR="$(mktemp -d)"` instead and reference `"$OUT_DIR/..."` in subsequent grep checks — the temp dir is cleaned up automatically and never touches the worktree. |
| A projection-verification step checks only one harness (e.g. Claude) or only a subset of the edited agents | Verify every edited agent × every generated harness directory (claude/, copilot/, codex/) with value-sensitive grep checks; for Codex `.toml` projections use `grep -E '^tools\s*='` for frontmatter checks and `grep -qF '<body text>'` for body-text propagation. |

# Health Disposition Storage Contract

## Purpose

Keep the human-facing ledger small without losing append-only provenance.

## Canonical Artifacts

- `docs/health/dispositions-history/YYYY/YYYY-MM.md`
  - append-only event log; one row per disposition event
- `docs/health/dispositions.md`
  - generated current-state projection; one row per latest disposition key
- `scripts/health_disposition_store.py`
  - the only supported parser, appender, and materializer

## Matching Key

`surface + dimension + object + finding`

Normalize only surrounding whitespace and internal whitespace runs.
Do not do fuzzy synonym matching in the storage layer.

## Rules

- Never append rows directly to `docs/health/dispositions.md`.
- All appends go to the appropriate month shard under `docs/health/dispositions-history/`.
- `docs/health/dispositions.md` is regenerated by `render_current_view()` after every append.
- Any step that needs historical closure evidence queries `iter_history_rows()`.
